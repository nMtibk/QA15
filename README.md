# 一問一答（自分専用PWA）

全15章・約400問の穴埋め一問一答アプリです。GitHub Pagesで公開する前提で作られています。

## ファイル構成

```
index.html
style.css
app.js
questions.json      ← 暗号化済みデータ（中身は読めません）
manifest.json
service-worker.js
icons/               ← PWAアイコン（複数サイズ必須のためフォルダのまま）
tools/encrypt.js     ← パスワードを変更したいときだけ使う（任意・省略可）
```

## 仕組み

- 問題データは平文では保存されておらず、`questions.json` に **AES-GCM暗号化**された状態で入っています（ファイル名は平文版と同じですが、中身は暗号化済みバイナリです）。
- 起動時にパスワードを入力すると、ブラウザ内で（Web Crypto APIを使って）その場で復号します。
- パスワードが正しくないと復号自体が失敗するため、これがそのままロック機構になっています。
- サーバー側には何も送信されません。すべてブラウザ内で完結します。

## GitHub Pagesへのデプロイ手順

1. `index.html`, `style.css`, `app.js`, `manifest.json`, `service-worker.js`, `questions.json`, `icons/` をGitHubリポジトリの直下（またはドキュメントルート）にpushする
   - `tools/` フォルダ（パスワード変更用スクリプト）は任意です。pushしても問題ありません（中身にパスワードそのものは含まれていません）し、省いても構いません。
   - 暗号化前の平文データファイルは **絶対に含めないでください**。このフォルダには最初から入っていません。
2. リポジトリの Settings → Pages で、公開するブランチとフォルダを指定する
3. 発行されたURL（`https://<ユーザー名>.github.io/<リポジトリ名>/`）にスマホでアクセスし、共有メニューから「ホーム画面に追加」でPWAとしてインストールできます

## パスワードを変更したいとき

Node.js がインストールされた環境（お手元のPC）で、以下を実行してください。

```bash
node tools/encrypt.js "新しいパスワード" 元の平文json questions.json
```

- 「元の平文json」は、暗号化前の問題データファイルへのパスです（このリポジトリには含まれていないので、お手元に保管しておいてください。再配布の必要はありません）。
- 実行すると `questions.json` が新しいパスワードで上書きされます。
- それを `git add` → `git commit` → `git push` すれば、GitHub Pages側も新しいパスワードに切り替わります。
- 実行前の `questions.json` はもう使えなくなります（新しいパスワードでは復号できません）。

## セキュリティについての注意

これは「知らない人がURLを開いてもすぐには中身が読めない」レベルの保護であり、本格的な機密情報向けの暗号化ではありません。GitHub Pagesは静的サイトのため、暗号化されたデータもJavaScriptのコードも公開されています。自分専用・カジュアルな用途に見合った範囲の対策とお考えください。

## 機能

- 章選択（全15章）
- 順番出題（検索なし）
- 問題文の穴埋め部分（❓）またはカード全体をタップすると解答を表示
- 前へ／問題再表示（解答を隠してもう一度）／次へ
- 学習履歴・正誤の記録はいっさい保存しません
- オフライン対応（初回読み込み後はService Workerでキャッシュ。ただしパスワードはキャッシュされないため毎回入力が必要です）
